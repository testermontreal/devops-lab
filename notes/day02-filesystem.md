⚡ Linux FHS — Cheatsheet (День 2)

"Where would you find application configuration files on Linux?" — это не вопрос памяти.
Это проверка того, понимаешь ли ты принцип разделения данных по изменчивости (principle of data mutability separation):

Неизменяемые двоичные файлы (immutable binaries) → /usr/bin, /usr/lib
Изменяемые конфиги (mutable configuration) → /etc
Изменяемые данные во время работы (runtime variable data) → /var
Временные данные (ephemeral/temporary data) → /tmp, /run
Виртуальные интерфейсы к ядру (kernel virtual interfaces) → /proc, /sys

# Полная карта FHS с interview-ready объяснениями

| Директория        | Назначение                                                   | Interview-ready ответ                                                                                                                                    |
| ----------------- | ------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| /                 | Корень (root directory)                                      | "The root of the entire filesystem tree. Everything in Linux is a file under /."                                                                         |
| /bin → /usr/bin   | Пользовательские команды (user binaries)                     | "Essential user-facing commands like ls, cp, bash. On modern systems /bin is a symlink to /usr/bin."                                                     |
| /sbin → /usr/sbin | Системные команды (system/admin binaries)                    | "Commands for system administration, typically run as root: fdisk, iptables, sshd."                                                                      |
| /etc              | Конфигурация системы (system-wide configuration)             | "/etc contains all system-wide configuration files. It's text-based, human-readable, and version-controllable. Think of it as the 'settings' directory." |
| /var              | Переменные данные (variable data)                            | "/var holds data that changes during normal operation: logs in /var/log, mail spools, package caches, Docker volumes."                                   |
| /tmp              | Временные файлы (temporary files)                            | "Temporary files cleaned on reboot. Never store anything important here. World-writable — a classic security concern."                                   |
| /home             | Домашние директории пользователей (user home directories)    | "Each user's personal space: ~ expands to /home/username. Configs, documents, everything user-specific."                                                 |
| /root             | Домашняя директория root (root user's home)                  | "The root user's home. Separate from /home intentionally — even if /home is on a failed mount, root can still log in."                                   |
| /proc             | Виртуальная ФС процессов (process virtual filesystem)        | "/proc is not on disk — it's a virtual filesystem generated by the kernel in real time. /proc/1/ is the init process, /proc/cpuinfo is CPU data."        |
| /sys              | Виртуальная ФС устройств (sysfs / device virtual filesystem) | "/sys exposes the kernel's view of hardware and drivers. More structured than /proc. Used for device management and tuning."                             |
| /dev              | Файлы устройств (device files)                               | "Device files: block devices like /dev/nvme0n1, character devices like /dev/null, /dev/random. In Linux, everything is a file — including hardware."     |
| /usr              | Unix System Resources                                        | "The bulk of the system: binaries, libraries, documentation, headers. /usr/bin, /usr/lib, /usr/share. Read-only after installation."                     |
| /opt              | Опциональное ПО (optional/third-party software)              | "Third-party software that doesn't follow standard FHS conventions installs here. ProtonVPN, JetBrains IDEs, custom enterprise tools."                   |
| /lib → /usr/lib   | Библиотеки (shared libraries)                                | "Shared libraries .so files used by binaries in /bin and /sbin. Think of them as DLLs in Windows."                                                       |
| /boot             | Загрузчик (bootloader files)                                 | "Kernel images, initramfs, GRUB config. On your system: EFI partition is /boot/efi on /dev/nvme0n1p1."                                                   |
| /mnt              | Точки монтирования (manual mount points)                     | "Temporary mount points for sysadmin use. On your system: /mnt/timeshift-sata and /mnt/shared (exFAT)." 12_fstab_mounts_blkid.txt​                       |
| /run              | Данные времени выполнения (runtime data)                     | "PID files, sockets, locks created since last boot. Cleared on every reboot. sshd stores its PID at /run/sshd.pid."                                      |
| /srv              | Данные сервисов (service data)                               | "Data served by the system: web files for Nginx would go in /srv/www. Often empty on desktops."                                                          |


# Навигация и исследование
ls -lah /etc           # все файлы с размерами
ls -laht /var/log      # по времени, свежие сверху
stat /etc/passwd       # полная метаинформация + timestamps
file /usr/bin/python3  # тип файла (ELF binary, script, etc.)
which git              # где находится бинарник
type ls                # alias, builtin или file?

# find — поиск файлов
find /etc -name "*.conf"              # по имени
find /var/log -size +10M              # большие файлы
find /etc -mtime -1                   # изменённые за 24 часа
find /tmp -perm -o+w -type f         # world-writable (security!)
find / -perm -4000 2>/dev/null       # SUID файлы
find /etc -type f -exec grep -l "ssh" {} \;  # файлы содержащие "ssh"

# Дисковое пространство
df -hT | grep -v tmpfs | grep -v loop  # реальные разделы
du -sh ~                               # размер домашней директории
du -h --max-depth=1 /var | sort -rh | head -10  # топ поддиректорий

# Текстовые инструменты
cat -n файл                    # с номерами строк
less /var/log/syslog           # постраничный просмотр
tail -f /var/log/auth.log      # live monitoring
head -20 /var/log/syslog       # первые 20 строк

# grep — поиск по содержимому
grep -rn "pattern" /etc/       # рекурсивно с номерами строк
grep -v "^#" файл | grep -v "^$"  # убрать комментарии и пустые строки
grep -E "error|warn" /var/log/syslog  # OR-паттерн
grep -c "pattern" файл         # посчитать совпадения
journalctl -b 0 -p err         # системные ошибки с последней загрузки

# Потоки (Streams)
команда > файл       # stdout в файл (перезаписать)
команда >> файл      # stdout в файл (добавить)
команда 2>/dev/null  # выбросить stderr
команда &> файл      # stdout + stderr в файл
cmd1 | cmd2          # stdout cmd1 → stdin cmd2
cmd | tee файл       # и в файл, и на экран
